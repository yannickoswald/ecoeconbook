Introduction#

‚ÄúIn computer programming, a variable is an abstract storage location paired with an associated symbolic name, which contains some known or unknown quantity of data or object referred to as a value; or in simpler terms, a variable is a named container for a particular set of bits or type of data (like integer, float, string, etc‚Ä¶)‚Äù - or so says Wikipedia cited from a proper programming book.

For our purposes as scientists and ecological economists, it is important to understand that variables help you store quantities that represent something about the real-world and you can do computations with them that provide you useful insights.

For example, in the previous sections we already encountered instances of variables. I say instance because they took on a concrete value - in our case the fact that the total primary energy consumption for the USA was 100 Exajoule.

let us abbreviate total primary energy consumption to TPEC, then we can define a variable

tpec_USA = 100 # unit is Exajoule (EJ)

And you just learned another very very important fact-of-life about programming: COMMENT YOUR CODE. A comment in python is the annotation after the symbol #. This is written into your code cell but not interpreted by the computer as code. It is very important however that others, and even yourself, can understand your own code. You will notice in time, when you come back a week later, to code you have previously written, you will be grateful to your past self for commenting adequately.

So before going through lots of different variables and data types to clarify what is ‚Äúvariable‚Äù about the tpec_USA above: We could theoretically do some computation with it say this is again the USA energy consumption for 2024 and we know this grows by 0.5% per year. Then we could make the following calculation to know about 2025.

tpec_USA = tpec_USA * 1.005

So we just used the variable tpec_USA to compute a new instance of itself. This is the kind of thing that really makes it ‚Äúvariable‚Äù.

Here is a list of the most common variable types

The int type is an integer, the float type is number that goes with decimal places, the boolean value stores a binary truth value and the string variable stores text basically.

Now we are moving to more complex data types. This means we will now treat tuples, lists, sets and dictionaries.

Tuples are fixed-size records. For typical use cases: coordinates, any immutable bundle.

Lists are ordered collections of all kinds of objects or variables and, in my experience, one of the or maybe the most important datatype for you to understand very well.

Another very important data structure is the dictionary (often abbreviated dict). A dictionary is a bit like list but the key difference (you will see this is a pun) here is that its structure does not derive from the order of elements within but from the fact that every element can be found by its key which you can freely choose. Technically in a list also every element has a key but they are just the number indices of its position and you cannot define them. If you are interested in going a bit deeper into the differences and commonalities you always can find cool programming knowledge on geekforgeeks.

An example dictionary holding energy consumption per person could look like the following for example:

# Dictionary: country ‚Üí energy consumption per person (GJ/person, realistic illustrative data)
energy_per_capita = {
    "Switzerland": 120.0,
    "Austria": 130.0,
    "Germany": 150.0,
    "United States": 300.0,
    "India": 25.0
}

# Example usage
print("Energy per capita in Austria:", energy_per_capita["Austria"], "GJ/person")

Energy per capita in Austria: 130.0 GJ/person

You cannot select element any longer in the familiar way like in a list. You can see in the following. This will throw an error message - another fact-of-life in programming you will have to become really familiar with.

This error specifically tells you it could not find the key ‚Äú0‚Äù. But you can do it this way by using an existing key.

A typology of energy units used in ecological economics and energy economics could be for example:

Common energy units#

  • Joule (J): SI base unit of energy.

    • \(1 \,\text{kJ} = 10^{3} \,\text{J}\)

    • \(1 \,\text{MJ} = 10^{6} \,\text{J}\)

    • \(1 \,\text{GJ} = 10^{9} \,\text{J}\)

    • \(1 \,\text{TJ} = 10^{12} \,\text{J}\)

  • Watt-hour (Wh): convenient for electricity.

    • \(1 \,\text{Wh} = 3600 \,\text{J}\) (because \(1 \,\text{W} = 1 \,\text{J/s}\) and \(1 \,\text{h} = 3600 \,\text{s}\)).

    • \(1 \,\text{kWh} = 10^{3} \,\text{Wh} = 3.6 \,\text{MJ}\)

    • \(1 \,\text{MWh} = 10^{6} \,\text{Wh} = 3.6 \,\text{GJ}\)

  • British thermal unit (BTU): common in some datasets, especially for fuels.

    • \(1 \,\text{BTU} \approx 1055.056 \,\text{J}\)

    • \(1 \,\text{MMBtu} = 10^{6} \,\text{BTU} \approx 1.055056 \,\text{GJ}\)

  • Tonne of oil equivalent (toe): aggregate energy benchmark.

    • \(1 \,\text{toe} = 41.868 \,\text{GJ} \approx 11.63 \,\text{MWh}\)

A first look at pandas DataFrames#

So far we have mostly worked with single values, lists, and dictionaries.
In real data work, we often want to handle tables: rows and columns, similar to working in Excel.

For this, we use the library pandas, and its main data type: the DataFrame. This dataframe is extremely important - it is basically the bread and butter of Python Data Scientist and of Quantitative Ecological Economists of course! This is also why the last chapter of this Introduction to Python will actually be about analysing a dataframe with actual data! But let us start easy and small.

1. Creating a small DataFrame#

We start from your dictionary of energy use per person:

import pandas as pd  # standard shortcut: "pd"

energy_per_capita = {
    "Switzerland": 120.0,
    "Austria": 130.0,
    "Germany": 150.0,
    "United States": 300.0,
    "India": 25.0
}

# Turn this into a DataFrame:

df_energy = pd.DataFrame({
    "country": list(energy_per_capita.keys()),
    "energy_GJ_per_person": list(energy_per_capita.values())
})

print(df_energy)

         country  energy_GJ_per_person
0    Switzerland                 120.0
1        Austria                 130.0
2        Germany                 150.0
3  United States                 300.0
4          India                  25.0

Each row is a country, each column is a variable.

Now it is important that you understand that dataframes are efficient because you subset and slice data, that is find it and take it out - even when dataframes are very large. Later you might work with dataframes that have thousands or even millions of rows.

2. Accessing a column#

Columns are often accessed like this:

# Get the whole column as a Series
col = df_energy["energy_GJ_per_person"]
print(col)

0    120.0
1    130.0
2    150.0
3    300.0
4     25.0
Name: energy_GJ_per_person, dtype: float64

You can also do:

df_energy.energy_GJ_per_person  # works if the column name has no spaces

0    120.0
1    130.0
2    150.0
3    300.0

If you want several columns:

subset = df_energy[["country", "energy_GJ_per_person"]]

3. Accessing rows with .loc and .iloc#

There are two common ways to select rows in a DataFrame:

  • .loc ‚Üí select rows by label (index name)

  • .iloc ‚Üí select rows by position (row number)

# First row (position 0)
print(df_energy.iloc[0])

# First three rows (positions 0, 1, 2)
print(df_energy.iloc[0:3])

country                 Switzerland
energy_GJ_per_person          120.0
Name: 0, dtype: object
       country  energy_GJ_per_person
0  Switzerland                 120.0
1      Austria                 130.0
2      Germany                 150.0

If you want to use country names as row labels, set the index:

df_energy = df_energy.set_index("country")
print(df_energy)

               energy_GJ_per_person

Now we can use .loc with country names:

# Single row by label
print(df_energy.loc["Austria"])

# Several rows by label
print(df_energy.loc[["Austria", "India"]])

austria_energy = df_energy.loc["Austria", "energy_GJ_per_person"]
