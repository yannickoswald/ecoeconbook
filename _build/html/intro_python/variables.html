📖 Introduction#

“In computer programming, a variable is an abstract storage location paired with an associated symbolic name, which contains some known or unknown quantity of data or object referred to as a value; or in simpler terms, a variable is a named container for a particular set of bits or type of data (like integer, float, string, etc…)” - or so says Wikipedia cited from a proper programming book.

For our purposes as scientists and ecological economists, it is important to understand that variables help you store quantities that represent something about the real-world and you can do computations with them that provide you useful insights.

For example, in the previous sections we already encountered instances of variables. I say instance because they took on a concrete value - in our case the fact that the total primary energy consumption for the USA was 100 Exajoule.

let us abbreviate total primary energy consumption to TPEC, then we can define a variable

tpec_USA = 100 # unit is Exajoule (EJ)

📖 Basic data types#

So before going through lots of different variables and data types to clarify what is “variable/can change” about the tpec_USA above: We could theoretically do some computation with it, say this is the USA energy consumption for 2024 and we know this grows by 0.5% per year. Then we could make the following calculation to infer about 2025.

tpec_USA = tpec_USA * 1.005

A note on the “Growth Factor” logic. Multiplying by 1.005 is the same as a 0.5% increase because you are adding the growth to the original amount:

  1. The Original (100%): Represented as 1.0.

  2. The Growth (0.5%): Represented as 0.005 (moving the decimal two places left).

  3. The Result: \(1.0 + 0.005 = 1.005\).

Quick Reference:

  • 1.05 = 5% growth

  • 1.005 = 0.5% growth

  • 1.0005 = 0.05% growth

Think of the ‘1’ as the anchor that keeps your current value, and everything after the decimal as the extra bit growing on top.

We will learn more about this later.

So we just used the variable tpec_USA to compute a new instance of itself. The previous value has been replaced by the newer value. This is the kind of thing that really makes it “variable”.

We could also have an additive process which might be easier for now such as

tpec_USA = tpec_USA + 5

Here is a list of the most common variable types

42,                        # int: number of households sampled
36.5,                      # float: emissions in MtCO₂    
is_active = True,          # bool: policy is active   
"Austria",                 # str: country name

What other value can the variable is_active assume?

The int type is an integer, the float type is number that goes with decimal places, the boolean value stores a binary truth value and the string variable stores text basically.

You can always check the type of variable this way with the type function (we learn what a function is later).

is_active = True
type(is_active)

bool

📖 Combining basic data types#

Now we are moving to more complex data types. This means we will now treat tuples, lists, sets and dictionaries.

Tuples are fixed-size records. For typical use cases: coordinates, which is admittedly less important throughout this course but still really good to know about this data type!

Lists are ordered collections of all kinds of objects or variables and, in my experience, one of the or maybe the most important basic datatype for you to understand very well.

📖 Dictionaries#

Another very important data structure is the dictionary (often abbreviated dict). A dictionary is a bit like a list but the key difference (you will see this is a pun) here is that its structure does not derive from the order of elements within, but from the fact that every element can be found by its key which you can freely choose. Technically, in a list also every element has a key but they are just the number indices of its position and you cannot define them. If you are interested in going a bit deeper into the differences and commonalities you always can find cool programming knowledge on geekforgeeks.

An example dictionary holding energy consumption per person could look like the following for example:

# Dictionary: country → energy consumption per person (GJ/person, realistic illustrative data)
energy_per_capita = {
    "Switzerland": 120.0,
    "Austria": 130.0,
    "Germany": 150.0,
    "United States": 300.0,
    "India": 25.0
}

# Example usage
print("Energy per capita in Austria:", energy_per_capita["Austria"], "GJ/person")

Energy per capita in Austria: 130.0 GJ/person

You cannot select element any longer in the familiar way like in a list. You can see this in the following. This will throw an error message - another fact-of-life in programming you will have to become really familiar with.

energy_per_capita[0]

---------------------------------------------------------------------
KeyError                            Traceback (most recent call last)
Cell In[9], line 1
----> 1 energy_per_capita[0]

KeyError: 0

This error specifically tells you it could not find the key “0”. But you can do it this way by using an existing key.

energy_per_capita["Switzerland"]

120.0

  1. Transform 1 million kWH and 1000 MWh to Terajoule and store the conversion factors as variables.

Start to carefully right everything you need e.g. start with 1*10 ** 6 for one kWh and times a million (it does not change the calculation but helps for clarity). Break down steps e.g. first transform 1 million kWh to Joule, then only to Terajoule.

The conversion factors you can find by summing all the power-numbers, so the powers of ten and everything you multiply you sum the power of ten number and everything you divide, you would subtract.

Example: To convert 2 million kWh to Terajoules (TJ), multiply your base numbers (\(2 \times 3.6\)) and simply add or subtract the powers of ten. Add the exponents when multiplying (+6 for million, +3 for kilo, +3 for Wh to Joules) and subtract when dividing (-12 for Tera), which gives \(7.2 \times 10^{(6+3+3-12)}\), leaving exactly 7.2 TJ.

  1. Make a dictionary that holds the conversion factors with appropriate keys.

  2. Print the entries from the dictionary and store the numeric values in a list instead.

📖 Common energy units list#

  • Joule (J): SI base unit of energy.

    • \(1 \,\text{kJ} = 10^{3} \,\text{J}\)

    • \(1 \,\text{MJ} = 10^{6} \,\text{J}\)

    • \(1 \,\text{GJ} = 10^{9} \,\text{J}\)

    • \(1 \,\text{TJ} = 10^{12} \,\text{J}\)

  • Watt-hour (Wh): convenient for electricity.

    • \(1 \,\text{Wh} = 3600 \,\text{J}\) (because \(1 \,\text{W} = 1 \,\text{J/s}\) and \(1 \,\text{h} = 3600 \,\text{s}\)).

    • \(1 \,\text{kWh} = 10^{3} \,\text{Wh} = 3.6 \,\text{MJ}\)

    • \(1 \,\text{MWh} = 10^{6} \,\text{Wh} = 3.6 \,\text{GJ}\)

  • British thermal unit (BTU): common in some datasets, especially for fuels.

    • \(1 \,\text{BTU} \approx 1055.056 \,\text{J}\)

    • \(1 \,\text{MMBtu} = 10^{6} \,\text{BTU} \approx 1.055056 \,\text{GJ}\)

  • Tonne of oil equivalent (toe): aggregate energy benchmark.

    • \(1 \,\text{toe} = 41.868 \,\text{GJ} \approx 11.63 \,\text{MWh}\)

# ==============================================
# Energy unit conversions: kWh / MWh → TJ
# ==============================================

# 1) Calculate conversion factors (basic)

# 1 million * Kilo (10^3) * Wh-to-Joule (3.6 * 10^3)
one_million_kWh_in_joule = 1 * (10**6) * (10**3) * 3.6 * (10**3) 
one_million_kWh_in_terajoule = one_million_kWh_in_joule / (10**12)

# 1000 * Mega (10^6) * Wh-to-Joule (3.6 * 10^3)
one_thousand_MWh_in_joule = 1000 * (10**6) * 3.6 * (10**3)
one_thousand_MWh_in_terajoule = one_thousand_MWh_in_joule / (10**12)

KWH_TO_TJ = 3.6 * (10**-6)  # 1 kWh in TJ, explanation: 1 kWh = 3.6 MJ = 3.6 * 10^6 J, and 1 TJ = 10^12 J, so 1 kWh in TJ is (3.6 * 10^6) / (10^12) = 3.6 * 10^-6
MWH_TO_TJ = 3.6 * (10**-3)  # 1 MWh in TJ, explanation: 1 MWh = 3.6 GJ = 3.6 * 10^9 J, and 1 TJ = 10^12 J, so 1 MWh in TJ is (3.6 * 10^9) / (10^12) = 3.6 * 10^-3  

# 2) Store conversion factors in a dictionary

conversion_dict = {
    "kWh_to_TJ": KWH_TO_TJ,
    "MWh_to_TJ": MWH_TO_TJ
}

# 3) Print results directly

# --- New section to prove equivalency ---
print("--- Energy Equivalency Check ---")
print("1 Million kWh in TJ:", one_million_kWh_in_terajoule)
print("1000 MWh in TJ:     ", one_thousand_MWh_in_terajoule)
print("Are the quantities mathematically equal?", one_million_kWh_in_terajoule == one_thousand_MWh_in_terajoule)
print("--------------------------------\n")
# ----------------------------------------

print("This is conversion factor 1: ", conversion_dict["kWh_to_TJ"], "TJ per kWh")
print("This is conversion factor 2: ", conversion_dict["MWh_to_TJ"], "TJ per MWh")

# Store numeric values of conversion factors in a list
values_list = [conversion_dict["kWh_to_TJ"], conversion_dict["MWh_to_TJ"]]
print("Numeric values list:", values_list)

Solutions explained in words

Scenario A: 1 Million kWh to Terajoules

Let’s break this down step-by-step. To keep things clear, we will start with the 1 from our 1 million kWh and separate our base numbers from our powers of ten.

Step 1: Convert to Joules Multiply the amount (1 million) by the prefix (kilo, so time one thousand) and the Wh-to-Joule conversion. \(1 \times 10^6 \times 10^3 \times (3.6 \times 10^3)\)

Group the base numbers (\(1 \times 3.6\)) and simply add the exponents (\(6 + 3 + 3 = 12\)): \(3.6 \times 10^{12} \text{ Joules}\)

Step 2: Convert to Terajoules

To go from Joules to Terajoules, divide by the Tera factor (\(10^{12}\)). Dividing means we subtract 12 from our exponent: \(3.6 \times 10^{(12 - 12)} = 3.6 \times 10^0\) Since \(10^0\) is exactly 1, our final answer is 3.6 TJ.

Scenario B: 1000 MWh to Terajoules

We will follow the exact same logic.

First, let’s write our starting amount of 1000 as \(1 \times 10^3\).

Step 1: Convert to Joules

Multiply the amount (1000) by the prefix (Mega, so one million) and the Wh-to-Joule conversion.

\(1 \times 10^3 \times 10^6 \times (3.6 \times 10^3)\)

Group the base numbers (\(1 \times 3.6\)) and add the exponents (\(3 + 6 + 3 = 12\)): \(3.6 \times 10^{12} \text{Joules}\)

Step 2: Convert to Terajoules

Just like before, divide by the Tera factor by subtracting 12 from the exponent: \(3.6 \times 10^{(12 - 12)} = 3.6 \times 10^0\)

Our final answer is 3.6 TJ.

Notice how adding the exponents proves that 1 million kWh and 1000 MWh represent the exact same amount of energy!

📖 A first look at pandas DataFrames#

So far we have mostly worked with single values, lists, and dictionaries.
In real data work, we often want to handle tables: rows and columns, similar to working in Excel.

For this, we use the library pandas, and its main data type: the DataFrame. This dataframe is extremely important - it is basically the bread and butter of Python Data Scientist and of Quantitative Ecological Economists of course! This is also why the last chapter of this Introduction to Python will actually be about analysing a dataframe with actual data! But let us start easy and small.

1.📖 Creating a small DataFrame#

We start from your dictionary of energy use per person:

import pandas as pd  # standard shortcut: "pd"

energy_per_capita = {
    "Switzerland": 120.0,
    "Austria": 130.0,
    "Germany": 150.0,
    "United States": 300.0,
    "India": 25.0
}

# Turn this into a DataFrame:

df_energy = pd.DataFrame({
    "country": list(energy_per_capita.keys()),
    "energy_GJ_per_person": list(energy_per_capita.values())
})

print(df_energy)

         country  energy_GJ_per_person
0    Switzerland                 120.0
1        Austria                 130.0
2        Germany                 150.0
3  United States                 300.0
4          India                  25.0

Each row is a country, each column is a variable.

Now it is important that you understand that dataframes are efficient because you subset and slice data, that is find it and take it out - even when dataframes are very large. Later you might work with dataframes that have thousands or even millions of rows.

A word on import

You may have noticed at the beginning of the code above, the presence of this line of code :

import pandas as pd

If this sounds to you completely unfamiliar, that’s completely normal, no need to panic. Python (as all languages) is built with several functionalities, that you can imagine as simple Lego bricks, so that every programmer can use. And so do you. But, most of the time, this isn’t enough for a non-basic usage. So people have developed more advanced functions, using these simple bricks, while also being easy-to-use for everyone.

This is what we called earlier a library, and this is exactly what Pandas is. But Python needs to know that you want to use a library for this task, and which one.

This is the job of the word import that notifies Python of this, as they are not included by default.

3. 📖 Accessing rows with .loc and .iloc#

There are two common ways to select rows in a DataFrame:

  • .loc → select rows by label (index name)

  • .iloc → select rows by position (row number)

# First row (position 0)
print(df_energy.iloc[0])

# First three rows (positions 0, 1, 2)
print(df_energy.iloc[0:3])

country                 Switzerland
energy_GJ_per_person          120.0
Name: 0, dtype: object
       country  energy_GJ_per_person
0  Switzerland                 120.0
1      Austria                 130.0
2      Germany                 150.0

If you want to use country names as row labels, set the index:

df_energy = df_energy.set_index("country")
print(df_energy)
